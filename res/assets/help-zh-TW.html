 破解器有七種破解方式.

 1. 代碼破解 ( 不需要Xposed ) :- 這個(代碼破解)項，是破解程式的代碼，當你更新被破解的應用前，你應該首先選擇「取消破解」選項，否則會導致新版本無法安裝，. (在更新應用安裝後你必須用 破解器 再次執行破解).

 2. 代碼緩存破解 ( 不需要Xposed ) :- 這個(代碼緩存破解)項，是破解程式的緩存代碼，使用此選項，你可以輕鬆地更新被破解的應用，不需要在安裝新版本之前選擇「取消破解」選項，但重新安裝或更新應用時破解會被移除. ( 需要在更新應用之後重新執行破解 ).

 3. 授權驗證破解 ( 需要Xposed ) :- 這個( 授權驗證破解 )主要是破解 Google 授權驗證和應用內購機制，在系統啟用 Xposed 的情況下，採取假數據作為來自Google服務器的「授權有效」的消息. 這個破解非常有效，但是會讓應用開發者很難受. ( [在破解之前，你需要首先運行應用至少一次]，只需要破解一次，在應用更新後仍然有效 ).

 4. 庫文件破解 ( 不需要Xposed ) :- 這個(庫文件破解)項，主要是破解應用的庫文件，一般是遊戲上的破解. ( 需要在更新應用之後重新執行破解 ).

 5. 數據儲存破解 ( 不需要Xposed ) :- 這個(數據儲存破解)項，主要在應用的數據儲存中進行更改，根據應用的需求，你有可能只需單獨應用此項破解，也可能需要與其它破解項配合使用. ( [在破解之前，你需要首先運行應用至少一次]，只需要破解一次，在應用更新後仍然有效 ).

 6. Xposed 掛鉤破解 ( 需要Xposed ) :- 這個( Xposed 掛鉤破解 )項，在啟用 Xposed 的情況下對應用進行動態的更改.（又一個讓開發者蛋痛的選項...） ( [ 只需要破解一次]在應用更新後仍然有效 ).

 7. 模組破解 ( 不需要Xposed ) :- 這個(模組破解)項，主要修復系統檢查機制，有些應用只需要執行這一項破解，但是有的保護比較完善的應用需要同時執行多項破解. ( 需要在更新應用之後重新執行破解 ).

 8. 生成補丁破解 ( 不需要Xposed ) :- 它是授權驗證破解的替代，但是可以在沒有 Xposed 的情況下工作,它使用系統預設的結果來破解，而不是像授權驗證破解一樣對授權驗證結果進行修改來破解它. ( [在破解之前，你需要首先運行應用至少一次]，只需要破解一次，在應用更新後仍然有效 ).

 註意 :- 如果某些應用需要多於一項破解，你只需應用優先級最高的一項（在應用詳請裡會顯示）. ( 優先級次序 :- 授權驗證破解 > Xposed 掛鉤破解 > 代碼破解 > 代碼緩存破解 > 庫文件破解 >模組破解 > 數據儲存破解 ).
破解有可能對應用的新版本無效，但是「破解」選項對於此破解的目標版本總是有效的.

 如果破解無效或者破解之後應用無法運行，你應該查看故障排解選項 ( 破解器 > 選單 > 故障排解 ) 並按照故障排查裡的內容去做. 如果仍然無效，請轉儲日誌 ( 破解器 > 選單 > 轉儲日誌) 然後把收集到的所有的文本文件發電郵給我 ( 破解器 > 選單 > 關於 > 發送電郵 ).
