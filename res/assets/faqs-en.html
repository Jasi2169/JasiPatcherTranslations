FAQs
Q1 :- Why I Can't Update Patcher, It Says Application Not Installed ?
A1 :- Please Uncheck 'Hide Patcher' Option In Patcher's Settings And Try Again.

Q2 :- After Enabling License Hook I Get Internet Not Found Error In Target App ?
A2 :- (License Hook Removed In v3.0) Try To Run Play Store And Press Home Button And Now Force Stop The Target App And Then Launch It.

Q3 :- After Enabling License Emulation The App Is Still Not Licensed ?
A3 :- Enable Target App In The 'Select Apps' Option In Emulation Fragment, If You Do Not Have Xposed Then This Option Can Not Be Clicked So Use 'Support Emulate' Patch From Universal Patch Fragment To Target App And Now Force Stop The App And Launch The App.

Q4 :- After Enabling Billing Emulation I Do Not Get Fake Purchase Pop Up In Target App ?
A4 :- Enable Target App In The 'Select Apps' Option In Emulation Fragment, If You Do Not Have Xposed Then This Option Can Not Be Clicked So Use 'Support Emulate' Patch From Universal Patch Fragment To Target App And Now Force Stop The App And Launch The App And Now While Buying The Pop Up Will Come.

Q5 :- Billing Pop Up Come But When I Purchase, It Does Not Reflect The Purchase ?
A5 :- Some Apps Require You To Relaunch Them So Force Stop Or Clear App From Recent And Now After Launching It Again They Should Be In Bought State.

Q6 :- Some Of The Apps Does Not Work With Root Emulation But With Xposed Emulation ?
A6 :- Emulation With Root Has Limitation Because To Work It Depends On Target Application's Code Where As In Xposed Emulation Nothing Is Required From Target App, If It Does Not Work Then Most Probably Patcher Is Unable To Inject In Target App So Use Xposed Emulation Because It's Just A Quality.

Q7 :- It Says Classes.dex N1 Patched And Classes2.dex N1 Failed But App Is Patched, How ?
A7 :- Patch Has Custom Patterns For Example N1 And N2 Are Two Different Patterns, It Is Not Necessary N1 Will Be In All Classes.dex, So It Should Say N1 Patched At Least Once In Any Of The Multi Classes.dex Present In App, It Can Be Found In All Classes.dex Or Some Of Them Depend On App So What Matters Different Patterns N1 Or N2 Both Should Say Patched At Least Once In Result No Matters How Many Multi Classes.dex Are Present In App, You Can Try Run The Target App To See The Outcome.

Q8 :- It Says Patch For Version v1.0.0 - v2.0.0 In Custom Patch But When Try Patching On v1.2.0 It Failed ?
A8 :- It Means The Patch Supplied In Custom Patch For Target App Is Tested On Old Version v1.0.0 And Latest Version v2.0.0 So It May Not Be Possible Every Time It Will Patch Middle Of Versions Or Older Than v1.0.0 Or Newer Than v2.0.0, It May Work Or May Not But It Should Work On Those It Is Tested With For Sure.

Q9 :- Fake Billing Pop Up Does Not Come Even After Force Stopping The App ?
A9 :- Try To Turn Internet On Because Some Apps Detect If Internet Is Off And If Off Then Does Not Send Request For Buying.

Q10 :- When Launching Patcher I Get Black Screen ?
A10 :- In Your Root Management App Make Grant Root As Default Setting, Once Patcher Gets Root Access You Can Change Root Management App's Settings Back To Previous One.
